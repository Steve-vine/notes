The code already uses "workspace" for the browser-style window tabs (ADR 0021). The Create Workspaces milestone introduces a user-facing Workspace concept (desktop-style note canvas), so the internal tab store is renamed to "tabs" before that lands, leaving one "workspace" concept in the codebase.

**Agreed plan (2026-07-09):**

- [ ] File renames (`git mv`): `workspace.svelte.ts` → `tabs.svelte.ts`, `workspaceStorage.ts` → `tabsStorage.ts`, `WorkspaceTabs.svelte` → `TabStrip.svelte`, `workspaceStorage.test.ts` → `tabsStorage.test.ts`.
- [ ] Symbol renames: `workspace` store → `tabs`, `WorkspaceTab` → `Tab`, `WorkspaceState` → `TabsState`, `WORKSPACE_KEY` → `TABS_KEY`, `serializeWorkspace`/`reviveWorkspace` → `serializeTabs`/`reviveTabs`. (`defaultTab`, `newTabId`, `ViewMode` already tab-centric — unchanged.)
- [ ] Update importers: `Main.svelte` (store + component usage, ~10 comments), `ViewTabs.svelte` (type import), `keymap.ts` labels ("New/Next/Previous workspace tab" → "… tab").
- [ ] localStorage key migration `notuvia:workspace` → `notuvia:tabs`: on load, read the new key, fall back to the old one if absent (rewritten under the new key on first save; old key left in place for rollback). `VERSION` stays 1 — shape unchanged.
- [ ] Aria-labels in `TabStrip.svelte` ("Workspace tabs" → "Tabs", "New workspace tab" → "New tab"); `brief/ui.md` wording. ADR 0021 untouched (append-only) — PR notes the terminology rename.

**Verification:** full `vitest run` + `npm run check`; manual — existing `notuvia:workspace` layout survives the upgrade, tab add/close/rename/cycle and shortcuts still work.

**Blocks:** the Workspace view brief (DEV-908).
