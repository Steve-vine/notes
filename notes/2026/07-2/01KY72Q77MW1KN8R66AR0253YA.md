## Context

`src/lib/TaxonomySettings.svelte` (~1,000 lines) is an invisible two-mode state machine: `draft === null` shows the taxonomy list, anything else swaps the whole pane to the editor. There's no breadcrumb or back affordance — the only exits from the editor are Cancel/Save at the bottom, and Escape is silently swallowed to protect unsaved work (DEV-708), which reads as "Escape is broken" rather than "you have unsaved changes".

## Change

Make the two levels explicit within the new settings shell (DEV-957):

* **List view**: keep the "Your taxonomies" / "System taxonomies" split, but tighten the rows to the app's list-row convention — name, scope chip, value count; delete stays two-click.
* **Editor view**: header with a back affordance ("← Taxonomies" or breadcrumb) plus the taxonomy name, so it's obvious where you are and how to get out.
* **Unsaved changes**: navigating back (or pressing Escape) with a dirty draft asks — a small confirm ("Discard changes?") instead of silently eating the key. Clean drafts navigate back immediately on Escape.
* Keep the `handleEscape()` / `isEditing()` contract with the Settings shell working.

## Acceptance

* From the editor it is always visible which taxonomy you're editing and how to return to the list.
* Escape in the editor: dismisses an open popover first; then, if dirty, prompts to discard; if clean, returns to the list; from the list, closes Settings.
* No way to lose edits without an explicit discard.
