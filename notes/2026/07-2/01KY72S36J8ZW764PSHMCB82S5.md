## Context

The Shortcuts section lists only the pane keymap (`ACTION_ORDER` in `src/lib/keymap.ts`). The New Note capture hotkey — Alt+Space — is registered as an OS-level global shortcut hardcoded in the Tauri backend (`src-tauri/src/lib.rs`, `Shortcut::new(Some(Modifiers::ALT), Code::Space)`), so it appears nowhere in Settings and users can't discover or confirm it.

## Change

Show it: a "Global" group at the top of the Shortcuts section with a read-only "New note — Alt+Space" row (help line explaining it opens capture from anywhere and is fixed for now), then the existing rows under a "Main window" group title.

Making the global hotkey remappable (persist in config, re-register live, handle OS conflicts) is out of scope — a follow-up issue if wanted.

## Acceptance

* The Shortcuts section shows the New Note global hotkey in the same visual style as the pane bindings, clearly non-editable.
