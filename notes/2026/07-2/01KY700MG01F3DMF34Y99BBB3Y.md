Write a new, append-only ADR (`decisions/00NN-rebrand-notula-to-notuvia.md`, next free number) recording the decision to rename the application from **Notula** to **Notuvia**, and — crucially — the **migration policy** every other issue in this milestone follows.

Per repo convention (`decisions/0001`), accepted ADRs are append-only and must not be edited to change meaning. Earlier ADRs keep saying "Notula"; this ADR supersedes the name going forward.

## What the ADR must capture

* **Decision:** rename to Notuvia across all code, config, and living documentation.
* **Migration policy** (chosen: full deep rename **with auto-migration**, one-time / idempotent / best-effort on first run) for each migration-sensitive identifier:
  * Default vault path `~/Notula` → `~/Notuvia` — auto-migrate existing folder.
  * Bundle identifier `uk.stevevine.notula` → `uk.stevevine.notuvia` — migrate the app-config dir (which stores the saved vault path) so the app doesn't forget the vault.
  * localStorage `notula:*` keys → `notuvia:*` — migrate-on-read so theme/layout/keymap survive.
  * `.notula/` index dir → `.notuvia/` — rebuildable (ADR 0003), migrate-or-rebuild.
  * `notula-attachment://` URI scheme → `notuvia-attachment://`.
* **Consequences:** historical ADRs retain the old name; repo itself is being renamed (see the repo-rename issue).

## Done when

* New ADR added in Date / Status / Context / Decision / Consequences format, status Accepted.
* No accepted ADR is edited in place to change meaning.
