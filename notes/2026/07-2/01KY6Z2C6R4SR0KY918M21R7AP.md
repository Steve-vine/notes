Let the user change the vault location after first run (ADR 0013). Independent of git — can land first.

The vault is chosen on first run; Settings must allow re-pointing it to another folder. Because the index is disposable (ADR 0003), re-pointing rebuilds the index from the new folder's files — no data move, no migration.

## Sub-steps

- [ ] Settings control to pick a new vault folder (native folder picker).
- [ ] On change: persist the new path in app config, point the store at it, and trigger a full index rebuild from the new folder.
- [ ] Adopt an already-populated folder (existing `notes/` + `taxonomies.yaml`) as well as an empty one.
- [ ] Restart/refresh file-watcher against the new path.
- [ ] Sensible handling if the chosen folder is invalid/unreadable (clear error, no partial switch).

## Definition of done

Vault can be moved to another folder from Settings; the app reindexes and operates against the new location; restart-safe.
