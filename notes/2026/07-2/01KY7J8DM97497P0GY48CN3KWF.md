Overlay each entity node on the estate **Dependency graph** (React Flow canvas, ISE-223/228) with its live signal/incident state, so an operator sees at a glance what on the estate is currently in trouble and can jump straight to it. Follow-up from ISE-228.

### Node overlay
- **Top-left of each node**, up to three small icons — the *same* tabler icons the left nav uses:
  - **Incident** — `IconAlertTriangle` (nav `/issues`)
  - **Alert** — `IconBellRinging` (nav `/alerts`)
  - **Observation** — `IconEye` (nav `/observations`)
- Each icon appears **only when that thing is currently present** for the entity.
- **Glow** (border/shadow) coloured by the **highest severity** across the present incident + alerts + observations: **Red = high, Orange = medium, Green = low, otherwise white/none**.
- Applies to the **root node and every neighbour node**.

### Click targets (confirmed with Steve 2026-07-23)
- **Alert / Observation** icons → open the **signal detail modal** in place (the existing `SignalDetail` component, as used on the Alerts/Observations screens) for that entity's open signal of that type. Where an entity has more than one open signal of a type, open the most severe (tie-break most recent).
- **Incident** icon → open the incident on **its own page**, `/issues/{incident_id}` (not a modal).
- On the pop-out graph window, the modal renders in that window; the incident navigation opens the incident page (same window is fine).

### "Present" means
- Signals: firing/open — `Finding.entity_id == e`, `resolved_at IS NULL`, not `ignored`/`silenced`.
- Incident: a non-`closed` `Issue` correlated to one of the entity's signals.

### Backend
- Extend `GET /entities/{id}/graph`: each `GraphNode` (and the root) carries the state it needs to drive the icons, links and glow — e.g. `{alert_finding_id, observation_finding_id, max_signal_severity, incident_id, incident_number, incident_severity}` (the finding ids are the most-severe open signal of each type, so the icon can open its `SignalDetail`; null when absent).
- Reuse the existing plumbing — `Finding.entity_id` (ISE-128) for the signal join, and `findings._live_incident_ids` (correlation_key → live incident) for the incident. One batched query over the reached entities; no N+1.

### Frontend
- Render the icon row + glow on the custom `EntityNode`; wire the click targets above. The Alert/Observation icons fetch the finding (`GET /findings/{id}`) to populate `SignalDetail`, or the graph carries enough to open it directly.
- Regenerate API types; add tests (backend query + node-styling helper).

Icons / links / glow spec confirmed with Steve 2026-07-23.
