Overlay each entity node on the estate **Dependency graph** (React Flow canvas, ISE-223/228) with its live signal/incident state, so an operator sees at a glance what on the estate is currently in trouble and can jump straight to it. Follow-up from ISE-228.

### Node overlay
- **Top-left of each node**, up to three small icons — the *same* tabler icons the left nav uses:
  - **Incident** — `IconAlertTriangle` (nav `/issues`) → links to the open incident, `/issues/{incident_id}`.
  - **Alert** — `IconBellRinging` (nav `/alerts`) → links to the entity's open alert(s).
  - **Observation** — `IconEye` (nav `/observations`) → links to the entity's open observation(s).
- Each icon appears **only when that thing is currently present** for the entity.
- **Glow** (border/shadow) coloured by the **highest severity** across the present incident + alerts + observations: **Red = high, Orange = medium, Green = low, otherwise white/none**.
- Applies to the **root node and every neighbour node**.

### "Present" means
- Signals: firing/open — `Finding.entity_id == e`, `resolved_at IS NULL`, not `ignored`/`silenced`.
- Incident: a non-`closed` `Issue` correlated to one of the entity's signals.

### Backend
- Extend `GET /entities/{id}/graph`: each `GraphNode` (and the root) carries `{open_alerts, open_observations, max_signal_severity, incident_id, incident_number, incident_severity}`.
- Reuse the existing plumbing — `Finding.entity_id` (ISE-128) for the signal join, and `findings._live_incident_ids` (correlation_key → live incident) for the incident. One batched query over the reached entities; no N+1.

### Frontend
- Render the icon row + glow on the custom `EntityNode`; wire the links (open in the same window; on the pop-out, respect that window).
- Regenerate API types; add tests (backend query + node-styling helper).

### Open decision (default chosen)
No per-signal detail route exists — the Alerts/Observations screens are lists, only Incidents have a per-item page. Default: **Incident** icon → the specific incident; **Alert/Observation** icons → the entity's detail page (`/estate/{entityId}`), where that entity's open signals are listed (the honest "actual entry"). If entity-scoped deep-links into the Alerts/Observations screens are wanted instead, that's a small add (a URL filter on `SignalsPage`) — confirm at build time.
