## Why

With headless git-sync mode (DEV-916), a Linux server with no desktop and no path to the laptop can run Claude Code against the vault. The setup deserves a documented recipe so it's reproducible per machine.

## What

Add a "Running on a headless server" section to `docs/mcp-server.md` covering:

* Clone the vault repo with a GitHub **deploy key** (per-machine SSH key with write access to just the vault repo) — auth stays delegated to the OS/git per ADR 0013.
* Prerequisites: system `git`, Rust toolchain; build with `cargo build --release -p notuvia-mcp` (notuvia-core has no platform-specific code).
* Register with Claude Code: `claude mcp add notuvia -- /path/to/notuvia-mcp --vault ~/notuvia-vault --git-sync --sync-interval 60`.
* Freshness expectations: laptop edit → GitHub in ~4–10s (app's debounced push); GitHub → server within the pull interval while an MCP session is live, plus a pull at MCP startup; server is simply stale between sessions.
* Conflict behaviour: keep-both conflict copies (DEV-615), surfaced in the app UI.
* Note that encrypted note bodies stay sealed over MCP (title/metadata only).

## Acceptance

Following the doc from scratch on a clean Linux box yields a working round-trip: a task created by Claude on the server appears in the laptop app, and vice versa.
