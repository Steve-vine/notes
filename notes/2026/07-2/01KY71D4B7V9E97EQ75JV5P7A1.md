    **Planned work (Claude, 2026-07-07):** a Duplicate button in the note statusbar (copy icon, next to the bin); clicking creates the copy and opens it in the pane.

    Backend `duplicate_note(id) → new id`, following the restore-as-new pattern (`rekey_as_restored_copy`): fresh id, `created`/`updated` = now, title suffixed **" (copy)"**. Copy semantics:
    - **Copied:** body, type, taxonomy values, project link, due, milestone/sprint choices, blocked_by.
    - **Dropped:** `comments` (commentary on the original), `favourite`, a project's `identifier` + `next_task_number` (identifier is unique), `number` — a task in an identified project gets a **fresh number** from the project counter.
    - **Attachments:** the referenced files are copied to the new note's attachment folder and the body references rewritten, so deleting the original can't break the copy.
    - **Encrypted notes** duplicate verbatim (same envelope, same password opens it — the fixed-AAD choice in ADR 0028 makes this work). Caveat: attachment references inside a sealed body can't be rewritten, so those keep pointing at the original.
