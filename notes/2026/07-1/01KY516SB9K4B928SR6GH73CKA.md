    **ADR 0042** records the register and adds **Documents** to ADR 0027's capability set. **Migration 0045** adds `document` + `document_tag`, and seeds the new `summarise-document` AI task type on Haiku 4.5 (constraint change + seed follow the ISE-95 precedent, so nothing needs configuring by hand).

    **Confluence connector** declares Documents *only* — no sync slices, no action catalogue, permanently: ISE reads the wiki and has no write path to it. Page id is parsed from the URL (three shapes handled; a tiny link is refused at registration rather than 404ing on a scrape hours later), and storage format is rendered to prose at fetch — a model should pay for words, not for a wiki's macro soup.

    Decisions worth flagging:

    - **Tags are the only link.** There is deliberately no document→entity foreign key. A document's tags go through `tags.get_or_create`, so a runbook registered as `env:prod` reaches a host DataDog tagged `env:Prod` — both resolve onto the same canonical. The link follows the estate instead of rotting against a host id.
    - **The content hash overrules the version token.** A version bump with identical text is a re-render, not an edit. Without that, every no-op save in Confluence would buy a paid re-summary — and change-driven summarising is the register's entire cost model.
    - **A 404 and a timeout are different facts.** `DocumentGone` → `gone` (row kept, last copy kept, no longer offered as current); anything else → `error`, which the next sweep recovers from on its own. Conflating them would either withdraw a runbook because Confluence blipped, or keep serving a deleted page.
    - **A URL nothing can fetch still registers.** The human act stands, and the row says why — better than refusing and losing the registration.
    - `read_document` is a new assist tool; summaries + freshness ride into `investigation_context` for free, the page itself is a deliberate pull.
    - `CitationEntity` gained `document`, so a chat answer can cite a runbook.

    **UI:** a Documents register under its own nav entry (register / edit / re-read / remove, with unreachable ones called out at the top and untagged ones flagged as reaching nothing), plus a Documents card on entity detail that renders nothing when no document reaches that entity.

The fourth source made real (Canon: "The Document Register"). Confluence is the source of truth; ISE scrapes it.

- **ADR (next free NNNN)**: the Document Register + the **Documents** connector capability (extends ADR 0027's capability set).
- **Model + migration**: document rows — URL, plain-text description, tag list, scraped content, AI summary, source last-modified, fetch status/first-seen/last-fetched.
- **Confluence Integration Type** declaring Documents (+ credential spec: base URL + API token; health check). Fetch renders page storage format → text.
- **Scheduled scrape** on a slow cadence (settings-configurable, obs-loop-style gating). Change detection via source version/last-modified; on change, regenerate the **AI summary** (short, structured; the only AI cost, and only on change).
- **Tag-resolved linking**: the document's tag list resolves through the Tag Dictionary to entities/groups — no manual entity links.
- **Consumption**: summaries of linked documents ride into `investigation_context` (with age presented honestly — "last changed 8 months ago"); a `read_document` tool pulls full text on demand.
- **UI (DoD)**: register/edit a document (URL, description, tags) and a Documents card on entity/group detail (title, description, freshness, link out to Confluence). Registration surface: a Documents section — modest page under its own nav or on the estate; decide in implementation with the nav IA in mind.
- Tests: scrape idempotency; change detection; tag-linking; context injection with age; fetch-failure handling (page gone → flagged, see extraction task for claim handling).
