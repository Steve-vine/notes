The fourth source made real (Canon: "The Document Register"). Confluence is the source of truth; ISE scrapes it.

- **ADR (next free NNNN)**: the Document Register + the **Documents** connector capability (extends ADR 0027's capability set).
- **Model + migration**: document rows — URL, plain-text description, tag list, scraped content, AI summary, source last-modified, fetch status/first-seen/last-fetched.
- **Confluence Integration Type** declaring Documents (+ credential spec: base URL + API token; health check). Fetch renders page storage format → text.
- **Scheduled scrape** on a slow cadence (settings-configurable, obs-loop-style gating). Change detection via source version/last-modified; on change, regenerate the **AI summary** (short, structured; the only AI cost, and only on change).
- **Tag-resolved linking**: the document's tag list resolves through the Tag Dictionary to entities/groups — no manual entity links.
- **Consumption**: summaries of linked documents ride into `investigation_context` (with age presented honestly — "last changed 8 months ago"); a `read_document` tool pulls full text on demand.
- **UI (DoD)**: register/edit a document (URL, description, tags) and a Documents card on entity/group detail (title, description, freshness, link out to Confluence). Registration surface: a Documents section — modest page under its own nav or on the estate; decide in implementation with the nav IA in mind.
- Tests: scrape idempotency; change detection; tag-linking; context injection with age; fetch-failure handling (page gone → flagged, see extraction task for claim handling).
