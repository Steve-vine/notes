**Bug (found on prod 2026-07-20).** On an incident detail, "Looks related to" and "Seen N times before" produce nonsense matches for DataDog incidents — e.g. "Low Disk Space is Warn" proposed as related to "[Synthetics] callroutingtwilio - Call Routing is Alert" and recalled as a prior of "Chinwag - UK - Application Exception is Alert".

**Root cause:** both mechanisms fall back to **same system + same `kind`** when the signal has no resolved estate entity — and every DataDog alert has kind `monitor_alert`, so the fallback degenerates to "any other DataDog monitor alert matches". DataDog signals currently *never* have an entity (see companion bug: DataDog contributes no estate entities), so the degenerate fallback is the permanent path:

- Merge proposals: `merge.py:propose_merges` (~:42, `Finding.kind == finding.kind` / reason "same signal kind")
- Recall: `recall.py:trigger_signature` (~:31) and `recall_prior_incidents` (~:90, `Finding.kind == finding.kind` + system fallback)

**Fix:** for Alerts, key the no-entity fallback on the signal's fine-grained stable identity instead of `kind` — the **monitor id**, already present in the correlation identity (`source_key` = `monitor/{id}/{group}`; use the `monitor/{id}` prefix, dropping the group). "Same monitor, different group/occurrence" is a meaningful related/seen-before match; "same universal kind" is not. Observations keep kind-based matching (their kinds are genuinely discriminating: crashloop, oom_kill…).

Notes:
- Matching stays deliberately loose (ADR 0029 — over-recall is cheap, both surfaces are human-gated); this fix removes the degenerate case, it does not tighten the philosophy.
- Companion task (same sprint): DataDog service-catalogue entity discovery — once DataDog signals resolve to entities, the entity-scoped primary match takes over and this fallback becomes the rare path. Both fixes are worth having: the fallback still governs signals whose entity isn't discovered yet.
- Tests: same monitor different group matches; different monitors same system do not; K8s observation matching unchanged.
