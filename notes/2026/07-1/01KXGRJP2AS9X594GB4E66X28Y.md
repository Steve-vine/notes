**Agreed approach (plan-mode, 2026-06-14):**

**Models** (mixins from db/mixins.py)

* `Assessment` (UUID/Timestamp/Actor/SoftDelete + CompanyScopedMixin): `core_control_id` FKтЖТcore_controls RESTRICT, `applicable` bool, `applicability_justification`, `status` enum (not_implemented/partial/implemented/not_applicable), `maturity_level` int 0тАУ5 null, `owner_id` FKтЖТusers null, `notes`, `evidence_links` JSONB list[str], `reviewed_at`/`next_review_at`. **Partial unique** (company_id, core_control_id) WHERE deleted_at IS NULL тАФ one current per pair.
* `AssessmentRevision` (append-only): `assessment_id`, `revision` int, snapshot of every field тАФ trend/history.
* Migration 0006 (autogenerateтЖТhouse-style; alembic check).

**API** (writes admin/editor; reads any authed)

* `PUT /assessments` тАФ upsert by {company_id, core_control_id}: create-or-update the current assessment + append a revision. 404 unknown company/control.
* `GET /assessments?company=<id>` (+ status/owner/maturity/domain/control filters); `GET /assessments/{id}`; `GET /assessments/{id}/revisions`.

**Tests:** upsert create+update (one row/pair, revisions increment), filters, revisions order, maturity 0тАУ5 (422), 404, role gates (viewer 403, unauth 401).

**Resolved forks:** (1) upsert-by-pair (PUT) not classic POST/PATCH; (2) Gap auto-raise deferred to <issue id="d4405a69-bde0-4406-9a48-861b062926d4" href="https://linear.app/stevevine/issue/DEV-403/gaps-model-api-and-view">DEV-403</issue>.
