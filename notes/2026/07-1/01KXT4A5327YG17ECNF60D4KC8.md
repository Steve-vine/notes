### Record Types
- Observation - A detected anomaly
- Alert - An alert triggered by an integration
- Incident - The investigation of related Alerts and/or Observations

### Obs Loop
The recurring observation check that run on a regular basis. Can be configured in settings, each integration can be included or opted out, and the schedule can be set per integration. 

The Obs Loops checks each integration for unusual or anonymous behaviour and records an ‘Observation’ in ISE.  

### Observations
Observations are deduplicated based on event and affected infrastructure, but a record is maintained for each observation detailing when it was observed. This provides a history or the observation.
Observations have a status field:
- New - Detected for the first time.
- Recurring - Has happened before within the last x days (Observation lifetime can be set in settings).
- Recovered - The observation went away on it’s own or was not seen again.
- Resolved - Has been fixed.
- Ignored - Known observation, don’t show this again.

### Alerts
Alerts are triggers from external monitoring systems, which have determined that a pre-defined failure state has been detected.  Alerts are created in ISE with the details provided by the integration. Alerts are deduplicated based on alert name but a record of occurrence (last triggered) is maintained within the history of the alert.
Alerts have a status field:
- Triggered - A triggered alert
- Recurring - Has Triggered before within the last x days (Alert lifetime can be set in settings).
- Recovered - The alert stopped alerting on its own.
- Resolved - Has been fixed
- Ignored - Known false alert, record the occurrence but leave the alert in Ignored state

### Incidents
Upon detection of an alert, or upon manual triggering by the user based on an observation, ISE will open an Incident and begin a targeted investigation of the affected systems, adding relevant observations or related alerts to an incident.

The Incident forms the timeline of what happened, who did what and when.
Incidents have a status field:
- New - A newly created incident
- Active - In incident that has been assigned to a person to deal with.
- Resolved - An incident that is believed fixed, it could be reactivated if it occurs again.
- Reactivated - An incident that was resolved but has been detected again.
- Closed - An incident that has been Resolved for more than x days without reoccurring automatically moves to closed (x can be set in settings).
