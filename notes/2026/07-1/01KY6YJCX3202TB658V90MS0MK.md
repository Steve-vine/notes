The backend for in-app taxonomy management (ADR — DEV-547). No UI here; that's the next brief.

**Checklist**

- [ ] Serialise the user (`system: false`) taxonomies to `taxonomies.yaml` as canonical YAML, written atomically (temp + rename). System taxonomies are never written (ADR 0009).
- [ ] Commands to **create / update / delete** a user taxonomy and its value pool (id, name, multi, allow_user_values, scope, values[] with id/label/order/colour?/is_done?). Validate (unique ids, non-empty, not a system id, valid scope).
- [ ] **Hot-reload:** after a write, rebuild the in-memory `TaxonomyRegistry` and reindex affected notes (the registry currently loads once in `VaultRuntime::start`). The vault watcher should also reload on an external `taxonomies.yaml` change.
- [ ] Self-write suppression so the app's own taxonomy write doesn't bounce (mirror the note self-write guard, DEV-534).
- [ ] Rust tests: round-trip (write → reload → equal), reject invalid/duplicate/system ids.

**Done when:** user taxonomies can be created/edited/deleted via commands, persisted to `taxonomies.yaml`, and reflected live in applicable-taxonomy lists + browse without a restart. Implements DEV-547. Builds on the DEV-483 loader.
