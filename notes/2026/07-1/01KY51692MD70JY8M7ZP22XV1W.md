Make human-asserted edges cheap (Canon: authority tier 1). The API already exists (`POST /entities/{id}/edges`, all five `EDGE_TYPES`, `enrichment.assert_edge`) — it has no UI at all today.

- "Add relationship" on the entity detail page: edge type picker (the five types, with plain-language labels — "depends on", "routes to"…), direction, target entity search (reuse the existing entity search), confirm.
- Remove: delete an *asserted* edge (add the DELETE endpoint if missing — verify); harvested/rule edges are not deletable here (they'd re-harvest — explain that in the UI).
- Provenance badges on graph edges (asserted / discovered / rule / proposed) so a reviewer can see why each edge exists — the verifiability principle applied to the graph view.
- Asserted edges are sticky per ADR 0028 §4; audited as today.
- Tests: assert/remove flows; provenance rendering.
