\<stack-name> Name of the VPC
\<region-code> Region to create the cluster in
\<cluster-role-name> The role used to run the EKS Cluster
\<cluster-name> The name of the EKS Cluster
\<node-role-name> The role used to run the EKS Nodes

#### 1. Create a VPC with private and public subnets
```Bash
aws cloudformation create-stack --region <region-code> --stack-name <stack-name> --template-url https://s3.us-west-2.amazonaws.com/amazon-eks/cloudformation/2020-10-29/amazon-eks-vpc-private-subnets.yaml
```
View progress in the CloudFormation dashboard - https://eu-central-1.console.aws.amazon.com/cloudformation

This is the Cloud formation yaml that will be run
![[amazon-eks-vpc-private-subnets 1.yaml]]

#### 2. Create the cluster IAM Role

Create a file called **eks-cluster-role-trust-policy.json** and add the following content
```Bash
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "eks.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

Create the role
```Bash
aws iam create-role --role-name <cluster-role-name> --assume-role-policy-document file://"eks-cluster-role-trust-policy.json"
```

Attach the required Amazon EKS managed IAM policy to the role
```Bash
aws iam attach-role-policy --policy-arn arn:aws:iam::aws:policy/AmazonEKSClusterPolicy --role-name <cluster-role-name>
```

#### 3. Create the EKS cluster

Open the EKS Console and choose the correct region
[https://console.aws.amazon.com/eks/home#/clusters](https://console.aws.amazon.com/eks/home#/clusters)

Choose **Add cluster**, and then choose **Create**

On the **Configure cluster** page, do the following:
Choose a name for the cluster
Select the Cluster Service Role created previously - \<role-name>
Click Next

On the **Specific Networking** page, do the following:
Select the VPC that was created in the previous steps - \<stack-name>
Click Next

On the **Configure logging** page, choose **Next**

On the Select Add-ons Page ensure that the following are selected:
Amazon VPC CNI
Kube-Proxy
CoreDNS
Click **Next**

On the Configure Selected Add-ons page choose **Next**

On the **Review and create** page, choose **Create**

Wait for the status to show **Complete**

#### 4. Setup kubectl

Retrieve kubeconfig file
```Bash
aws eks update-kubeconfig --region <region-code> --name <cluster-name>
```

#### 5. Create the Nodes IAM Role

##### Managed Nodes (EC2)
Create a node IAM role
Create a file called **node-role-trust-policy.json** and insert the following code
```Bash
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "ec2.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

Create the node IAM role
```Bash
aws iam create-role --role-name <node-role-name> --assume-role-policy-document file://"node-role-trust-policy.json"
```

Attach the required IAM Policies to the role
```Bash
aws iam attach-role-policy --policy-arn arn:aws:iam::aws:policy/AmazonEKSWorkerNodePolicy --role-name <node-role-name>
```

``` Bash
aws iam attach-role-policy --policy-arn arn:aws:iam::aws:policy/AmazonEC2ContainerRegistryReadOnly --role-name <node-role-name>
```

``` Bash
aws iam attach-role-policy --policy-arn arn:aws:iam::aws:policy/AmazonEKS_CNI_Policy --role-name <node-role-name>
```

#### 6. Create the Nodes

Open the EKS Console
[https://console.aws.amazon.com/eks/home#/clusters](https://console.aws.amazon.com/eks/home#/clusters)
Select the cluster
Navigate to **Compute tab**
Choose **Add Node Group**

On the **Configure Node Group** page, do the following:
For **Name**, enter a unique name for your managed node group.
For **Node IAM role name**, choose \<node-role-name> created in a previous step
Click **Next**

On the **Set compute and scaling configuration** page, accept the default values and choose **Next**.

On the **Specify networking** page, accept the default values and choose **Next**.

On the **Review and create** page, review your managed node group configuration and choose **Create**.

After several minutes, the **Status** in the **Node Group configuration** section will change from **Creating** to **Active**.
